Course Notes Contributor Guide

This guide explains how to create new content pages that match the "Gray Style" and static architecture of the course website.

1. Start with a Template

Don't start from scratch. Use one of the two templates in notes/pages/:

  • TEMPLATE_PREAMBLE.html: Best for introductory pages, high-level overviews, or simple text-heavy content.
  • TEMPLATE_NOTES.html: Best for technical chapters. It includes section headers, exercises, and math examples.

2. HTML Structure (The "Block" System)

The notes use a nested system to keep layout consistent. Every piece of content should be wrapped in a block-container.

Plain Text / Content Block

<div class="block-container">
    <div class="block block-plain-kind block-gram">
        <div class="atom-body-container">
            <div class="atom-body">
                <p>Your text here...</p>
            </div>
        </div>
    </div>
</div>

Section Header

<div class="block-container">
    <div class="block block-segment-kind block-section">
        <div class="atom-segment-kind">
            <div class="segment-display-num">1</div>
            <div class="col"><span>Section Title</span></div>
        </div>
    </div>
</div>

3. Exercises & Collapsible Solutions

To keep the page clean, solutions are hidden by default and require a click to toggle.

  1. Add block-collapsible block-collapsed classes to the solution block.
  2. Add onclick="toggleBlock(this)" to the anchor tag.
  3. Ensure ../notes.js is linked in the <head>.
<!-- Solution Example -->
<div class="block block-secondary-kind block-solution block-collapsible block-collapsed">
    <div class="atom-name-title">
        <a class="no-decoration" role="button" onclick="toggleBlock(this)">
            <span class="atom-name">Solution</span>
        </a>
    </div>
    <div class="atom-body-container">
        <div class="atom-body"> ... </div>
    </div>
</div>

4. Mathematical Content (KaTeX)

We use KaTeX for fast rendering. Always use the following delimiter configuration in your <head> to avoid backslash escaping issues:

delimiters: [
    {left: "$$", right: "$$", display: true},
    {left: "\\(", right: "\\)", display: false},
    {left: "\\[", right: "\\]", display: true}
]

5. Visual Standards (CSS)

  • Main Colors:
    • Accent Red: #CD5C5C (Titles, Labels)
    • Box Background: #f9f9f9 (Propositions, Definitions)
    • Dark Mode BG: #333333
  • Containers: Content is centered with max-width: 50em via base.css or 780px via atom.css.
Pro Tip: When adding images, use class="atom-img" to ensure they are centered and scale correctly.

6. Deployment Checklist

  • Link ../../base.css, ../chapter.css, and ../atom.css.
  • Link ../notes.js for toggles.
  • Update <title> and favicon.png link.
  • Ensure the top <nav> bar is present.
  • Verify all KaTeX math renders correctly (no raw \[ showing).
