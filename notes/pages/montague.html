Montague Grammar

1
The Type System

Richard Montague's central insight was that natural language is a mathematical object: every well-formed phrase has a unique semantic type, and sentences are built by combining types through function application — exactly like a programming language or the typed lambda calculus.

Before we can say what any word means, we need a type system to say what kind of thing its meaning is.

Atomic types

There are just two primitive types:

  • $e$ — the type of entities (individuals in the world: John, the Eiffel Tower, your left shoe).
  • $t$ — the type of truth values ($\top$ or $\bot$, i.e. True or False). A complete, declarative sentence denotes something of type $t$.

Functional types

Everything else is built by function formation: if $\alpha$ and $\beta$ are types, then $\alpha \to \beta$ is the type of a function from $\alpha$-things to $\beta$-things. This is exactly the arrow type you know from Haskell, OCaml, or any typed lambda calculus.

The arrow is right-associative: $\alpha \to \beta \to \gamma$ means $\alpha \to (\beta \to \gamma)$, a function that takes an $\alpha$ and returns a function from $\beta$ to $\gamma$. This is just currying. Some important derived types:

TypeReads asExample
$e \to t$function from entities to truth valuesa one-place predicate like sings, is a man: maps an entity to True iff it sings/is a man
$e \to e \to t$function from entities to $(e \to t)$ functionsa two-place predicate like loves: takes an object, returns a property of subjects
$(e \to t) \to t$function from $(e \to t)$ to truth valuesa generalized quantifier (GQ) like every man, some woman, John (lifted)
$(e \to t) \to (e \to t) \to t$takes a noun property, returns a GQa determiner like every, some, no

The key rule — Function Application (Rule B) — says: if $f$ has type $\alpha \to \beta$ and $x$ has type $\alpha$, then $f(x)$ has type $\beta$. Composing meanings is just applying functions.

Why types matter. Without types, you could accidentally ask whether a truth value sings, or apply a determiner directly to a verb. The type system makes these nonsensical combinations syntactically impossible — you get a type error before you ever reach the semantics.
2
Lambda Abstraction and Beta-Reduction

Montague semantics is written in the typed lambda calculus. If you've written any functional programming, this will feel familiar.

Lambda abstraction

$\lambda x. \phi(x)$ denotes the function that takes an input $x$ and returns $\phi(x)$. The variable $x$ is bound by the $\lambda$. So $\lambda x. \text{Sings}(x)$ is the function that maps any entity to True iff it sings — this is the denotation of the predicate sings, of type $e \to t$.

You can abstract over any type. $\lambda P. P(\text{john})$, where $P$ ranges over $e \to t$ functions, is the function that takes a property and returns True iff John has it. This is of type $(e \to t) \to t$.

Beta-reduction ($\beta$-reduction)

When you apply a lambda function to an argument, you substitute the argument for the bound variable everywhere in the body. This substitution is called $\beta$-reduction, written with the $\xrightarrow{\beta}$ arrow:

$$(\lambda x. \phi(x))(a) \xrightarrow{\beta} \phi(a)$$

Two concrete examples:

(λx. Sings(x))(john) →β Sings(john) (λx. λy. Loves(y, x))(mary) →β λy. Loves(y, mary) [a function from subjects to "y loves mary"]

Beta-reduction is the entire computational mechanism of Montague semantics. Composing a sentence means repeatedly applying and reducing lambda terms until you have a closed formula of type $t$ with no remaining lambdas.

Alpha-equivalence

The name of a bound variable is irrelevant. $\lambda x. \text{Sings}(x)$ and $\lambda y. \text{Sings}(y)$ are the same function ($\alpha$-equivalent). We'll silently rename variables to avoid clashes during substitution.

3
Primitives, Composition, and Type Lifting

We now assign denotations to a small vocabulary, annotated with types.

WordTypeDenotationGloss
john$e$$j$a specific entity
sings$e \to t$$\lambda x.\, \text{Sings}(x)$the property of singing
man$e \to t$$\lambda x.\, \text{Man}(x)$the property of being a man
every$(e \to t) \to (e \to t) \to t$$\lambda P.\, \lambda Q.\, \forall x\,(P(x) \to Q(x))$universal quantification

Deriving "every man sings"

We build meaning bottom-up from the syntax tree:

Step 1 — Form "every man" (Det + CN → GQ): every = λP. λQ. ∀x(P(x) → Q(x)) man = λx. Man(x) Apply every to man: (λP. λQ. ∀x(P(x) → Q(x)))(λx. Man(x)) →β λQ. ∀x(Man(x) → Q(x)) Type check: (e → t) → t ✓ (a GQ) Step 2 — Combine GQ with VP ("sings"): (λQ. ∀x(Man(x) → Q(x)))(λx. Sings(x)) →β ∀x(Man(x) → Sings(x)) Type check: t ✓ (a complete sentence)

The problem with "John sings"

Now try "John sings." Syntactically, it has the same structure: NP + VP. But john has type $e$, while "every man" has type $(e \to t) \to t$. The VP combinator can't accept both — we have a type mismatch.

Type Lifting (Rule A) resolves this. We lift the proper noun john from a mere entity $j : e$ up to a generalized quantifier of type $(e \to t) \to t$:

$$\text{john}_{\text{lifted}} = \lambda P.\, P(j)$$

This reads: "John is the set of all properties that John has." Now the derivation works uniformly:

(λP. P(j))(λx. Sings(x)) →β (λx. Sings(x))(j) →β Sings(j)

Both "John sings" and "Every man sings" are now derived by the same rule (GQ + VP). This uniformity is the whole point: the grammar has a single, general composition rule rather than special cases for proper nouns versus quantifiers.

Generalized Quantifier Theory. The lifted form $\lambda P. P(j)$ treats "John" as a quantifier: the trivial quantifier that ranges only over John. This is not just a formal trick — it reveals that proper nouns and determinative NPs are the same semantic category. Both denote sets of properties (equivalently, sets of sets of individuals). This unification is one of Montague's greatest contributions.
4
Intensionality and the Unicorn Problem

The framework so far treats noun phrases as denoting things (or sets of properties of things) that actually exist. But natural language can refer to non-existent objects:

John seeks a unicorn.

This sentence is perfectly grammatical and can be true — even though there are no unicorns. A naive analysis would assign "a unicorn" an empty extension and force the sentence to be trivially false (or undefined). That's clearly wrong.

Compare it with:

John finds a unicorn.

Here, if the sentence is true, a unicorn must} exist. Finding requires a real thing to be found. Seek does not. The contrast is called the distinction between intensional and extensional verb positions.

Denotation of "a unicorn" as a GQ

Using our GQ machinery, the determiner a has the denotation:

$$\text{"a"} = \lambda P.\, \lambda Q.\, \exists x\,(P(x) \land Q(x))$$

So "a unicorn" (type $(e \to t) \to t$) is:

$$\text{"a unicorn"} = \lambda Q.\, \exists x\,(\text{Unicorn}(x) \land Q(x))$$

This is the set of all properties $Q$ such that at least one unicorn has $Q$. If unicorns don't exist, this set is empty — but the function itself is still well-formed. The GQ exists even when its extension is empty.

Intensional verbs: seek

For seek, Montague's analysis keeps the object NP at its abstract GQ level. Seek doesn't demand that the NP resolve to an actual entity; it takes the GQ itself as its argument:

$$\text{Seek}(\text{john}, \underbrace{\lambda Q.\, \exists x\,(\text{Unicorn}(x) \land Q(x))}_{\text{the GQ "a unicorn"}})$$

John stands in the seeking relation to the concept of a unicorn. No existential commitment to actual unicorns is made. The formula is consistent with a world containing no unicorns at all.

Extensional verbs: find (via Meaning Postulates)

Find is different. When John finds something, that something is a real entity — not merely a concept. Montague handles this with a Meaning Postulate, a constraint on models that forces the object position of find to be resolved to an actual individual. The postulate is:

$$\text{Find}(u, NP) \iff NP\!\left(\lambda v.\, \text{Find}_{\text{ext}}(u, v)\right)$$

This says: "John finds [the GQ]" is true if and only if the GQ contains the property $\lambda v.\, \text{Find}_{\text{ext}}(\text{john}, v)$. Let's reduce this step by step for "John finds a unicorn":

Apply the NP "a unicorn" = λQ. ∃x(Unicorn(x) ∧ Q(x)) to the property from the postulate λv. Find_ext(john, v): (λQ. ∃x(Unicorn(x) ∧ Q(x))) (λv. Find_ext(john, v)) →β ∃x(Unicorn(x) ∧ (λv. Find_ext(john, v))(x)) →β ∃x(Unicorn(x) ∧ Find_ext(john, x))

The existential quantifier $\exists x(\text{Unicorn}(x) \land \ldots)$ is now explicit in the logical form. The math enforces what our intuition demands: if John finds a unicorn, there exists a unicorn that John found.

Intensional vs. extensional — the test. A verb is extensional in an argument position if substituting co-referring expressions preserves truth. If "Lex Luthor is Clark Kent's enemy" is true and find is extensional, then "Superman found Clark Kent's enemy" implies "Superman found Lex Luthor." For intensional verbs like seek, this substitution can fail — if Lois is seeking Superman's enemy without knowing who that is, she may not be seeking Lex Luthor in any psychologically meaningful sense.
5
Scope Ambiguity and Derivational History

Consider the sentence:

Every man loves a woman.

This sentence is genuinely ambiguous — not vague, but structurally ambiguous between two distinct logical readings:

  • Weak (surface) scope: For every man, there exists (possibly a different) woman he loves. $\forall z(\text{Man}(z) \to \exists y(\text{Woman}(y) \land \text{Love}(z,y)))$
  • Strong (inverse) scope: There is one particular woman that every man loves. $\exists y(\text{Woman}(y) \land \forall z(\text{Man}(z) \to \text{Love}(z,y)))$

These have different truth conditions. The weak reading is much weaker — it's true in almost any world with romantic activity. The strong reading makes a specific claim about one woman. Montague's account derives both readings from the same surface string by tracking derivational history — the order in which the syntax tree is assembled.

Derivation 1: weak scope (direct assembly)

Step 1 — Build VP "loves a woman": loves = λy. λx. Love(x, y) [type e → e → t] a woman = λQ. ∃y(Woman(y) ∧ Q(y)) [type (e → t) → t] Apply loves to a woman via QR: (λQ. ∃y(Woman(y) ∧ Q(y)))(λy. λx. Love(x, y)) →β ∃y(Woman(y) ∧ (λx. Love(x, y))) [type e → t — a property of subjects] More cleanly: λx. ∃y(Woman(y) ∧ Love(x, y)) Step 2 — Apply subject GQ "every man": (λP. ∀z(Man(z) → P(z)))(λx. ∃y(Woman(y) ∧ Love(x, y))) →β ∀z(Man(z) → ∃y(Woman(y) ∧ Love(z, y))) ✓ weak scope

Derivation 2: strong scope ("quantifying in")

To get the inverse scope reading, Montague uses a quantifying-in rule: we first insert a pronoun placeholder $\text{him}_1$ in the object position, build the sentence with this placeholder, and then bind it by the object GQ from outside.

Step 1 — Build "every man loves him₁": Treat him₁ as a free variable of type e. "loves him₁" = λx. Love(x, him₁) [type e → t] Apply subject "every man": (λP. ∀z(Man(z) → P(z)))(λx. Love(x, him₁)) →β ∀z(Man(z) → Love(z, him₁)) Step 2 — Quantify in "a woman", replacing him₁ with a bound y: (λQ. ∃y(Woman(y) ∧ Q(y)))(λy. ∀z(Man(z) → Love(z, y))) →β ∃y(Woman(y) ∧ ∀z(Man(z) → Love(z, y))) ✓ strong scope

The two derivations use the same lexical entries and the same composition rules. The ambiguity falls out of the order of operations — which GQ gets applied first. This is a clean, principled account of scope: no need for covert movement rules or additional representational levels.

Scope and processing. Psycholinguistic evidence shows that humans prefer weak scope (surface order) by default, with strong scope readings available but requiring more processing effort. This aligns naturally with Montague's derivational account: the direct assembly (Derivation 1) is computationally simpler.
6
Intensional Logic and Possible Worlds

Everything so far evaluates meaning at a single fixed world and time. But natural language routinely talks about possibilities, necessities, beliefs, and counterfactuals — contexts where truth can vary across worlds and times.

Montague's Intensional Logic (IL) adds two new dimensions of evaluation:

  • A set $W$ of possible worlds (ways things could have been).
  • A set $T$ of times (moments or intervals).

With this, every expression can be evaluated relative to an index $\langle w, t \rangle$. This gives us the key distinction:

  • Extension of $\alpha$ at $\langle w, t \rangle$: the value $\alpha$ actually takes in world $w$ at time $t$. Written $\vee\alpha$ (the "down" operator, also ${}^\vee$ or "ext"). For an individual concept like "the temperature," the extension at $\langle w, t \rangle$ is the actual temperature reading in that world at that moment.
  • Intension of $\alpha$: the function $\widehat{\alpha} : W \times T \to \text{extension}$, mapping every index to what $\alpha$ denotes there. Written $\wedge\alpha$ (the "up" operator). This is the concept behind $\alpha$ — what it means across all possible contexts.

The temperature paradox

Consider the sentence pair:

  1. "The temperature is 90."
  2. "The temperature is rising."

If we naïvely substitute the number 90 for "the temperature" in (2), we get "90 is rising" — nonsensical. The problem is that "is 90" and "is rising" make claims at different logical levels:

  • "is 90" evaluates the extension at the current index: ${}^\vee({}^\wedge\text{temperature}) = 90$.
  • "is rising" evaluates the intension as a function over time: $\text{Rising}({}^\wedge\text{temperature})$, asking whether the rate of change $\frac{d}{dt}[{}^\wedge\text{temperature}(w, t)]$ is positive.

You cannot substitute the extension into an intensional context — the type system blocks it. "90" has type $e$ (or a numerical type); "the temperature" in "is rising" has type $W \times T \to \text{number}$. These are different types; function application fails cleanly.

The Cain/Adam paradox and modal contexts

Consider: "Necessarily, the father of Cain is Adam."

In the actual world, the extension of "the father of Cain" is indeed Adam. So their extensions are identical. But Necessarily ($\Box$) quantifies over all possible worlds. Since there is a possible world where Adam was never born (and some other individual fathered Cain), the intensions of "the father of Cain" and "Adam" are not equal functions over $W \times T$.

This is the core of Frege's original puzzle about sense and reference, now formalized: two expressions can share an extension while having different intensions. The modal operator $\Box$ forces evaluation over intensions, not extensions, which is why the sentence comes out false (or at least not necessarily true) despite the actual-world identity.

The up/down operators ${}^\wedge$ and ${}^\vee$. You can think of ${}^\wedge$ ("up") as wrapping an expression inside a function, and ${}^\vee$ ("down") as forcing it open to get the current value. They are inverses: ${}^\vee({}^\wedge\alpha) = \alpha$ at any fixed index. This lets you explicitly control whether an expression is evaluated extensionally or kept as an intensional concept — the very control Montague's semantics provides.
7
Event Semantics: Aspect, Adverbs, and Thematic Roles

Our basic Montague lexicon for verbs treats sings as a simple predicate of entities. This immediately runs into trouble:

  • "John sings" (habitual: John regularly sings) vs. "John is singing" (progressive: a singing event is happening now).
  • "John walked quickly" — where should quickly attach? It seems to modify the verb, but the standard type of walks ($e \to t$) has no "slot" for manner adverbs.
  • "John buttered the toast in the kitchen with a knife" — do we want a separate predicate for every combination of location and instrument? That scales catastrophically.

Neo-Davidsonian Event Semantics (building on Donald Davidson's 1967 proposal) solves all three problems by introducing a new atomic type $v$ for events, alongside $e$ and $t$.

Verbs as event predicates

A verb no longer directly takes an entity as its argument. Instead, it denotes a predicate of events:

$$\text{"sings"} = \lambda e.\, \text{Singing}(e) \quad [\text{type } v \to t]$$

Participants are linked to events via thematic roles — predicates relating an event to a participant:

  • $\text{Agent}(e, x)$ — $x$ is the intentional instigator of event $e$
  • $\text{Theme}(e, x)$ — $x$ is the thing affected or moved by $e$
  • $\text{Instrument}(e, x)$, $\text{Location}(e, x)$, etc.

Progressive vs. habitual

  • Progressive — asserts the existence of a current, ongoing event: $$\text{"John is singing"}: \quad \exists e\,(\text{Singing}(e) \land \text{Agent}(e, j) \land \tau(e) \ni \text{now})$$ where $\tau(e)$ is the run-time of event $e$.
  • Habitual — uses a generic operator GEN over a regular pattern of events: $$\text{"John sings"}: \quad \text{GEN}[e]\,(\text{Singing}(e) \land \text{Agent}(e, j))$$ This does not assert that any particular singing event is happening now.

Manner adverbs and the conjunction property

Adverbs attach as conjuncts on the event variable. This is the key insight — it requires no new composition rules:

"John walked quickly in the park" ∃e (Walking(e) ∧ Agent(e, j) ∧ Quick(e) ∧ Location(e, park)) Each modifier simply ∧-conjoins a new condition on e.

This elegantly predicts the entailment patterns we observe. If "John walked quickly in the park" is true, then "John walked" must also be true — you just drop the conjuncts. In the old type-raising analysis with no events, deriving this entailment was impossible without stipulation.

Why not just use adverbs as VP modifiers? The alternative — treating quickly as a function $(e \to t) \to (e \to t)$ from VP-meanings to VP-meanings — fails to capture the entailments. If "walks quickly" is just some function $f$ applied to "walk", there's no guarantee that $f(\text{walk})(x) \to \text{walk}(x)$. The event-semantic account makes this entailment automatic and principled.
8
Advanced Paradoxes and Extensions of the Framework

The liar's paradox and Tarski's undefinability theorem

Consider: "This sentence is false."

In standard compositional semantics, this sentence cannot be constructed. Because meaning is built bottom-up, the subject "this sentence" would have to refer to the root node of the syntax tree — but the root node doesn't exist until the composition is complete. The self-reference fails at the compositional level before it can even become a paradox.

But we can engineer the paradox with Gödel numbering: encode sentences as natural numbers, find the number $n$ that encodes the sentence "the sentence with Gödel number $n$ is false," and write:

$$\text{"The sentence encoded as } n \text{ is false."}$$

Now the self-reference is indirect, going through arithmetic rather than syntax — and the machinery of Montague grammar cannot rule it out a priori.

The resolution is Tarski's Undefinability Theorem (a semantic corollary of Gödel's First Incompleteness Theorem, 1936). Tarski proved:

No sufficiently expressive formal language can contain its own truth predicate.

That is, if your language can express enough arithmetic, you cannot define a predicate $\text{True}(x)$ within that same language that correctly identifies the truths of the language. Any attempt produces a contradiction. Truth must always be defined in a strictly stronger metalanguage. In Montague grammar extended with Gödel numbering, $\text{True}$ and $\text{False}$ cannot be first-class predicates — they escape into the metalanguage, and the paradox never evaluates inside the system.

Hyperintensionality: the problem with logical equivalence

Possible worlds semantics has a structural flaw. Consider two tautologies:

  • $A$: "Green grass is green."
  • $B$: "White snow is white."

Both are true in every possible world. In standard intensional logic, two expressions that agree on all possible world/time indices are the same intension — the same function. So $A$ and $B$ have identical meanings. But they clearly don't mean the same thing! Similarly, "groundhog" and "woodchuck" corefer in every world, but "Phil believes groundhogs are cute" and "Phil believes woodchucks are cute" can have different truth values if Phil doesn't know they're the same animal.

This is the problem of hyperintensionality: some semantic contexts are too fine-grained for possible-worlds intensions to distinguish. Two main solutions have been proposed:

  1. Structured meanings / structured propositions. Instead of treating a proposition as a set of possible worlds, treat it as the actual syntactic structure of the logical formula — its parse tree, with constituent concepts as nodes. "Green(grass)" and "White(snow)" are different structures, so their meanings differ even though they agree on all worlds. This gives us Frege's Sinn as a real mathematical object.
  2. Impossible worlds. Expand the space of worlds to include impossible ones — worlds where logical laws fail, where "groundhog" and "woodchuck" denote different animals, where $2 + 2 = 5$. Epistemic operators like believe and know quantify over the agent's doxastic alternatives, which may include impossible worlds corresponding to the agent's false beliefs or incomplete knowledge. This allows Phil's belief worlds to contain worlds where groundhogs and woodchucks are distinct, explaining why the attitude reports can come apart.
The tradeoff. Structured meanings are compositionally very well-behaved but require a whole new theory of propositional structure. Impossible worlds preserve the possible-worlds framework and work naturally with dynamic and epistemic logics, but face a philosophical question: in what sense does an "impossible world" exist? Both approaches remain active research areas in formal semantics and philosophical logic.

Looking forward: Dynamic semantics and discourse

Montague grammar is almost entirely sentence-level. But natural language has crucial cross-sentence phenomena: pronouns can pick up antecedents from previous sentences ("John walked in. He was tired."), definite descriptions presuppose antecedents, and tense/aspect interlock across discourse. Discourse Representation Theory (DRT) and Dynamic Predicate Logic (DPL) extend Montague-style compositionality to handle these, by treating meaning as an update function on information states rather than a static truth condition. These frameworks build directly on everything in these notes — they keep the type theory and lambda calculus, and add dynamics on top.
